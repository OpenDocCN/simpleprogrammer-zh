# 如何开始一个成功的副业项目

> 原文：<https://simpleprogrammer.com/start-successful-side-project/>

你有没有觉得你只对一个特定的领域有信心？

好像你只知道软件宇宙中的一个小角落？然后，当你试图逃避它，去做一些不同的事情时，你会感到沮丧。

如果您需要从头开始构建一个完整的功能性项目，请使用:

1.  初始研究和机器设置(包装、分期付款)
2.  管理依赖关系(库、构建工具、自动化)
3.  整个业务逻辑加上高测试覆盖率
4.  生产部署

…那么除了你舒适区的小角落，其他人必须做所有的事情？

我知道我们很多人都有这种感觉，尤其是初级开发人员，他们加入了一个庞大、复杂的项目，其代码库已经开发了多年。你可能会觉得自己严重缺乏经验。

工作经验是一个微妙的问题。感觉好像每个人都夸大了他们对给定语言或框架的候选人要求的年数。

有时候是不现实的！看看这个:

![side project](img/25351871b167a100b04c12f0e34a4583.png)

虽然我怀疑这样的工作机会是否真的存在，但有一点是肯定的:如果你有经验，工作机会会不断向你涌来。如果你不是，选择是有限的。竞争是惊人的。

那么所有的初学者应该如何获得经验呢？

## 获得经验的可靠方法是开始一个副业项目。

兼职项目不一定是你在工作的同时开发的项目。它可以是你日常学习之外的任何[自我驱动的项目。这是你应用和发展技能的一种方式。](http://www.amazon.com/exec/obidos/ASIN/1524758841/makithecompsi-20)

最棒的是，你不会仅仅为了练习某项技能而做单独的、脱离上下文的、编造的练习。您正在创建作为整体的**解决方案**。你正在解决的问题类似于真实项目中的问题，类似于你将来可能从事的工作。

我可以不停地谈论副业对你职业生涯的好处，尤其是在早期阶段。许多软件社区已经很好地理解了这一点，尤其是简单程序员的观众。

这篇文章是关于大多数人陷入困境的地方:选择一个主题并坚持到底。

每个人都知道他们应该拥有一个副业。然而，只有少数人设法让一个项目为他们工作。一个好的想法可以让你走上正确的道路，优化你的学习，发展你的思维，同时让你保持兴奋和好奇，以便走得更远。然后你最终会建立[你的投资组合来推动你的职业发展](https://www.myfirstitjob.com/powerful-portfolio-website/)。

然而，许多开发人员没有想到他们真正喜欢构建的项目。以我的经验，我注意到人们倾向于更关注他们锻炼了什么技能或框架，而不是项目主题。他们很快意识到随着时间的推移，这变得有点无聊——最终，他们永远不会坚持到底。

让我们来看看如何计划你的兼职项目的关键点，这样它就能完成，并给你获得你想要的职位的经验。

## 选择主题——关于什么的？

每个项目的目的都是为了解决一个问题，你也应该如此。你如何决定解决什么问题？换句话说，这个项目应该是关于什么的？

选择项目主题时，有两个关键步骤:

1.  选择正确的语言和框架
2.  让你觉得有趣

选择正确的语言和框架是容易的部分。

首先，确保你选择了一个利基市场(如果你还没有选择的话)。利基实际上就是你想成为的人——例如，web 开发人员、iOS 开发人员或 Java 开发人员。具体情况可能不同，例如，一个 web 开发人员意味着 HTML 加 CSS 加 JavaScript 加一些 JavaScript 框架。但是你也可以选择成为一名 Ruby on Rails 5 开发人员——这就更加具体了。

接下来，**找到与你的领域相关的库和工具**。最好的方法是在你的领域内寻找工作机会，并观察他们还需要什么样的经验。例如，如果你想成为 Node.js 开发者，也许你*也*需要学习 Express.js 和 MongoDB。如果您希望使用 Ruby on Rails，您可能还需要了解一些 SQL。

在这些网站上进行简单的搜索，你就可以在你的领域内找到合适的职位空缺:

*   [的确](https://www.indeed.com/)
*   [Craigslist](https://craigslist.org/)
*   [正宗的工作](https://www.authenticjobs.com/)
*   [调剂处理](https://devitjobs.uk/)
*   [骰子](http://www.dice.com/)

收集一份清单，列出你的职位需求中提到的所有库、框架、工具和软件。这是一个快速可靠的方法，保证你会花时间做一些你以后可能会专业做的事情。一旦知道了将要使用的库和工具集，就可以为项目选择主题了。

## 选择一个有趣的话题

![](img/a1e2391daac155eadf2b52124af4dd54.png)

首先，你需要知道你的编程领域有能力做什么。例如，如果你只想专注于 HTML 和 CSS，你将无法创建一个动态的 web 应用程序。你应该关注一个静态网站。或者，如果您是 C++开发人员，您可能希望远离 web 开发，专注于桌面应用程序。

然后，远离编程，想想你的个人兴趣。试着说出几个你最喜欢的话题和活动。你的爱好是什么？你在空闲时间喜欢做什么？ 

这些问题的答案可以成为你的副业主题的绝佳选择。

你喜欢另类摇滚吗？考虑构建一个你可以实际使用的音乐应用——有很多开源音乐应用编程接口(API)。喜欢桌游？为什么不自己想一个简单的游戏呢？更进一步——如果你是哈利波特系列的粉丝，就把它变成哈利波特主题，如果你很少能找到很多人一起玩，就把它变成双人游戏。

你是老板，你发号施令——试着在项目中加入尽可能多的主题或解决尽可能多的你自己的问题。在这个项目中寻求和表达自己。这会让你渴望并渴望用它实现更多。

看看你是否能建立一个简短的主题列表，让你迫不及待地开始编码。然后，您可以很容易地过滤掉超出您的编程范围的内容。希望你能留下一两个真正伟大的想法。

如果你仍然停滞不前，看看你喜欢使用的现有项目或应用程序。你可以借用你想要的任何想法，只要你是非商业性的。

例如，如果你是一个足球迷，你可以创建一个简单版本的国际足联经理游戏，选择球员和模拟游戏。如果你是纸牌游戏的爱好者，你可以创建自己的游戏。如果你发现很难组织你的一天，你可以创建一个基本的 Evernote 原型。

仔细思考一个你个人感兴趣的话题可能需要一点时间，但是绝对值得。选择一个你感兴趣的主题可能会导致一个展示你技能的项目，因为你会自然地在你喜欢的事情上投入更多的时间和激情。

如果你也在寻找一个兼职项目——一个有可能为你赚钱的项目——那么约翰的《兼职项目创意》是必读之作！

## 项目的规模

项目应该有多大？这是规划项目的一个非常重要的部分。

试着从小处着手。计划一些你粗略估计在持续工作的一个月内准备好的事情。**甚至两周。**

从小处着手是个好主意，原因有三:

1.  你提前完成了一个功能性项目。这可以防止你感到不知所措，并提高你的热情。获得早期胜利并保持积极的势头至关重要，而且更有趣。
2.  您随时可以在以后添加新功能。想一想您最常用和喜爱的一些应用程序。什么类型不重要。你能快速推荐一个新的很酷的功能吗？如果有的话，你会*喜欢的。我敢打赌，如果你真的考虑五分钟，你至少会有三个。而且这是我们说的全功能大 app！想象一下，对于你的闪亮的小项目，你会有多少想法。*
3.  人们在计划时通常会低估。如果你计划在一个月内完成，很有可能在第二个月末，你将几乎完成*。编程通常都是这样的。意想不到的困难发生得早，也经常发生；偶尔会有 bug，配置可能会很麻烦，或者有时候代码需要重构。还有，你需要[应对拖延症](https://www.myfirstitjob.com/beating-procrastination/)。记住，当你计划你的项目时，你的想法通常是你最兴奋的时候。兴奋和“我等不及要开始了！”感觉是低估的常见原因。*

## *项目的复杂性*

*![](img/2f538848245b41c080ffd7ba4bde9992.png)*

*根据大多数开发人员的经验(以及我自己的经验)，我建议您将项目建立在您已经掌握的技能的基础上，但至少留下同样多的“未知”部分。*

*在你的舒适区安全起步是很重要的，因为它让你走上正确的轨道，并为未来的挑战树立信心。但是尝试新的概念，自己研究，经历“试错”才是你真正学习和成长最多的地方。如果你想从你的项目中获得最大收益，这两者之间的平衡是关键。*

*努力达到这个比例:*

1.  *40%——我已经做过并且知道如何去做的事情*
2.  *40%——我还没有做过的事情，需要对自己进行研究*
3.  *20%——我甚至不确定是否有可能做到的事情，但我很想核实一下*

*[How to aim when choosing a topic for your side project](https://infogram.com/e12b21ae-ebdf-45e9-a93a-dc8cab740487)
[Infogram](https://infogram.com)*

*目标是确保你有足够的舒适空间，但也包括足够的好奇心和不确定性。*

*请记住，这是你的个人项目，与你工作中的项目不同，你可以控制它的发展。在我职业生涯的早期，我总是把目标定得太高，最终，事情超出了我的能力范围。如果你发现自己也在这样做，那就退一步，坚持自己的优势。*

*另一方面，我总是喜欢在建设项目的每一步中至少有一些挑战。即使你重复以前做过几次的动作，看看你是否能多做一点。有没有比第一个想到的更好的方法或者库？这次你能更好地组织你的代码吗？*

*问自己这些问题可以确保你不会只是重复你已经知道的东西，而是将目标定得更高，并最终提高更多。作为一名软件开发人员，获取新概念的速度和准确性对您的职业生涯至关重要。那些成功的人是那些把学习变成习惯的人。*

*你的个人项目是养成这种习惯的绝佳场所——并且乐在其中！*

## *一旦你准备好了，充分利用你的项目*

*记住，一旦你完成了它，你可以利用你的副业作为你职业生涯的杠杆。“你造了什么？”是每个开发人员面试的首要问题之一。通常，最终的“是”或“否”取决于你在回答问题时表现出的能力水平。*

*想象一下向面试官展示你正在运行的应用程序的影响。解释发生了什么问题，你如何选择解决它们，为什么。*

*也许你有你工作中最喜欢的部分，一些让你感到自豪的东西。一个优雅的解决方案，应用设计模式、算法、元编程、与 API 的集成，或者仅仅是一个好的接口。你可以挑选印象最深刻的部分，用它来展示你的技能。*

*毕竟，你不只是完成了一门课程或做了一个练习。您从头开始创建了一个功能项目，研究并配置了依赖项，设计了一个架构，并用代码解决了一个实际问题。这表明你不仅是一个精通的开发人员，还是一个善于分析的思考者，组织良好，有能力成为一个功能项目的唯一驱动力。*
